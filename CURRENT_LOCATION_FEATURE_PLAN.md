# 현재위치 아이콘 기능 방안 보고서

**요구사항**: 오른쪽 하단 축척 표시 위에 "현재위치" 아이콘을 두고, 터치 시 **그 순간의 현재 위치**를 지도에 표시.

---

## 공통 전제

- **Geolocation API**: `navigator.geolocation.getCurrentPosition()` 또는 `watchPosition()` 사용.
- **권한**: 브라우저/OS에서 위치 권한 허용 필요. 거부 시 사용자 안내(alert 등) 필요.
- **표시 위치**: 뷰어 UI에서 축척 표시(`.scale-display`) **위쪽**에 고정 배치(예: `bottom: 70px` 또는 축척과 간격 두고 배치).
- **동작**: 아이콘 터치 시 → 위치 획득 → 지도에 해당 위치 표시(및 필요 시 지도 중심 이동).

---

## 방안 1: 단순 마커 1회 표시 (getCurrentPosition 1회)

### 내용

- 터치 시 `getCurrentPosition()` **한 번** 호출.
- 성공 시:
  - 해당 위·경도에 **마커 1개** 추가(예: 파란 점 또는 Google Maps 기본 "현재 위치" 스타일).
  - 지도 **중심**을 해당 좌표로 이동(선택적으로 줌 레벨 유지 또는 일정 줌으로 설정).
- 이전에 표시한 "현재 위치 마커"가 있으면 **제거 후 새 마커만** 표시(동시에 하나만 유지).

### 장점

- 구현이 단순하고 코드량이 적음.
- **배터리·리소스** 부담이 거의 없음(요청 시 1회만 위치 조회).
- "지금 이 순간 내 위치"를 한 번 확인하는 용도에 잘 맞음.
- 기존 DXF·사진·텍스트 기능과 독립적으로 추가 가능.

### 단점

- **지속 추적 없음**. 사용자가 이동해도 마커는 갱신되지 않음. 다시 터치해야 최신 위치가 표시됨.
- 정확도가 낮을 때(실내 등) 사용자가 "갱신"하려면 다시 터치해야 함.

### 구현 포인트

- 전역 변수: `currentLocationMarker` (Google Maps Marker 또는 Circle). 표시 시 setMap(map), 새로 표시 전 기존 것 setMap(null).
- 옵션: `getCurrentPosition`에 `{ enableHighAccuracy: true, timeout: 10000, maximumAge: 0 }` 등으로 "터치 순간의 새 위치" 위주로 조회.

---

## 방안 2: 정확도 반경(원) 함께 표시

### 내용

- 터치 시 `getCurrentPosition()` 1회 호출. 성공 시 **위·경도 + accuracy(미터)** 를 받음.
- 지도에:
  - **마커**: 현재 위치(중심).
  - **원(Circle)**: 반경 = `accuracy` 로 "위치가 이 원 안에 있을 수 있다"는 것을 시각적으로 표시.
- 지도 **중심**을 해당 좌표로 이동하고, **줌**은 원이 화면에 잘 보이도록 자동 조절(예: `fitBounds`로 원의 bounds에 맞춤).

### 장점

- 사용자가 **위치 정확도**를 직관적으로 이해할 수 있음(실내·GPS 불량 시 원이 크게 나옴).
- 도면 작업 시 "대략 이 범위 안에 있다"는 판단에 유리함.
- 전문적인 지도 앱과 비슷한 UX.

### 단점

- **구현량 증가**: Circle 오버레이 추가, accuracy 단위(m) → 지도 상 거리 변환, fitBounds 로 줌 조절 등.
- accuracy가 매우 크면(수 km) 원이 지나치게 커져 화면을 가릴 수 있음 → 상한(예: 최대 반경 500m) 적용 필요.
- 방안 1보다 코드·테스트가 다소 복잡함.

### 구현 포인트

- `google.maps.Circle` 사용, `center`, `radius: position.coords.accuracy`, `map`, 스타일(반투명 채우기 등) 설정.
- 기존 "현재 위치 마커"와 "현재 위치 원"을 함께 관리하고, 다음 터치 시 둘 다 갱신.

---

## 방안 3: 터치 시 1회 표시 + 짧은 시간 자동 갱신(옵션)

### 내용

- 터치 시 **처음에는** `getCurrentPosition()` 1회로 마커 표시 + 지도 중심 이동(방안 1과 동일).
- 추가로 **일정 시간(예: 30초) 동안만** `watchPosition()` 으로 주기적 갱신(예: 5초 간격 또는 `maximumAge` 활용).
- 갱신될 때마다 마커 위치를 새 좌표로 이동. 30초 경과 후에는 `clearWatch()` 로 추적 종료.

### 장점

- **움직이는 사용자**(걷기, 차량 등)에게 유리. 터치 한 번으로 잠깐 동안은 최신 위치가 반영됨.
- 무한 추적이 아니라 **제한 시간** 후 자동 종료되므로 배터리 부담이 watchPosition 무제한 사용보다는 적음.

### 단점

- **구현 복잡도**: watchPosition ID 저장, 타이머로 clearWatch 호출, "갱신 중" 상태 표시(선택) 등.
- **배터리**: 방안 1보다는 GPS/위치 사용이 길어짐.
- 사용자가 "한 번만 보고 싶다"는 경우에는 30초 갱신이 불필요할 수 있음.

### 구현 포인트

- `watchId = navigator.geolocation.watchPosition(...)` 저장.
- `setTimeout(() => { navigator.geolocation.clearWatch(watchId); }, 30000)` 로 30초 후 해제.
- 콜백에서 기존 currentLocationMarker 위치만 업데이트(`setPosition`)하면 됨. 지도 중심은 첫 1회만 이동할지, 매 갱신마다 이동할지 정책 결정 필요.

## 비교 요약

| 구분 | 방안 1 (단순 1회) | 방안 2 (정확도 원) | 방안 3 (1회 + 짧은 갱신) |
|------|-------------------|---------------------|---------------------------|
| 구현 난이도 | 낮음 | 중간 | 중간 |
| 배터리 | 매우 적음 | 매우 적음 | 적음~중간 |
| UX (한 번 확인) | 적합 | 적합(정확도 가시화) | 적합 |
| UX (이동 중) | 부적합(재터치 필요) | 부적합 | 적합 |
| 코드량 | 적음 | 중간 | 중간 |

---

## 권장

- **우선 적용**: **방안 1**으로 구현 후, 사용자 피드백에 따라 **방안 2**(정확도 원) 또는 **방안 3**(짧은 갱신)을 추가하는 단계적 접근을 권장합니다.
- "터치하는 순간의 현재 위치를 지도에 나타낸다"는 요구에는 방안 1이 가장 직접적으로 부합하며, 배터리와 유지보수 측면에서도 유리합니다.
