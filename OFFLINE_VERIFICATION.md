# 오프라인 동작 확인 보고

**확인 요청**: 최초 도면을 화면에 연 후, 오프라인에서도 모든 기능이 작동하는지 여부.

---

## 1. 네트워크 사용처 정리

| 구분 | 대상 | 네트워크 필요 | 비고 |
|------|------|----------------|------|
| **최초 1회** | Google Maps API 스크립트 | **필요** | `index.html`: `https://maps.googleapis.com/maps/api/js?key=...` |
| **최초 1회** | DXF 파서·변환·앱 스크립트 | 불필요 | `config.js`, `local-storage.js`, `libs/dxf-parser.min.js`, `dxf-to-geojson.js`, `app.js` 모두 로컬 |
| **도면 로드** | DXF 파일 읽기 | 불필요 | `<input type="file">` + `FileReader` (로컬) |
| **저장/불러오기** | 프로젝트·사진·텍스트 | 불필요 | IndexedDB (`local-storage.js`) |
| **사진 압축** | `fetch(dataUrl)` | 불필요 | data URL 대상 로컬 fetch (네트워크 미사용) |
| **내보내기** | ZIP/개별 다운로드 | 불필요 | Blob 생성 + `URL.createObjectURL` + `<a download>` (로컬) |
| **배경 지도** | "배경 없음" | **오프라인 가능** | 타일 요청 없이 스타일만 적용 (요소 숨김) |
| **배경 지도** | "도로/위성/하이브리드" | **오프라인 시 제한** | 줌/패닝 시 Google 타일 요청 발생, 캐시된 영역만 표시 가능 |

---

## 2. 결론

- **최초 1회**: 반드시 **네트워크 필요**. Google Maps API 스크립트를 불러와야 지도가 초기화됨.
- **도면을 화면에 연 후**:
  - **배경 지도를 "배경 없음"으로 사용하는 경우**:  
    도면 표시, 사진 추가/보기/저장/삭제, 텍스트 추가/편집/저장, DXF 문자 클릭 모달, 축척 표시, 전체보기/줌, 자료 내보내기(ZIP·개별) 등 **모든 기능이 오프라인에서 동작 가능** (로컬 파일·IndexedDB·Blob만 사용).
  - **배경 지도를 "도로/위성/하이브리드"로 사용하는 경우**:  
    줌/패닝 시 새 영역에 대한 **타일 요청이 발생**하므로, 오프라인에서는 **이미 캐시된 영역만** 표시되고, 새로 보이는 영역은 빈 칸 또는 로딩 실패가 날 수 있음. 도면 자체·사진·텍스트·저장·내보내기 등 앱 기능은 그대로 동작.

---

## 3. 요약

| 조건 | 오프라인에서 모든 기능 동작 여부 |
|------|----------------------------------|
| 최초 실행 (도면 열기 전) | ❌ Google Maps API 로드 필요 |
| 도면 연 후 + **배경 없음** | ✅ 예 (모든 기능 로컬 처리) |
| 도면 연 후 + **도로/위성/하이브리드** | ⚠️ 앱 기능은 동작, 배경 타일만 캐시된 범위 내에서만 표시 |

**정리**: “최초 도면을 화면에 열고 난 뒤”에는 **배경을 “배경 없음”으로 두면** 오프라인에서도 **모든 기능이 동작**한다. 배경 지도를 쓰는 경우에는 배경 타일만 네트워크 의존이 있고, 나머지 기능은 오프라인에서도 동작한다.
