# DXF 문자 표시·모달 기능 — 성능 영향 보고

## 변경 요약
- **문자(TEXT/MTEXT)**: Data 레이어 Point를 사진 마커처럼 작은 원으로 표시하되, 50% 작게(6px), 초록색.
- **동작**: 해당 원 터치 시 간단 모달로 텍스트 내용 표시. 모달 밖 터치 시 모달 닫기.

## 성능 영향 분석

| 항목 | 내용 | 부하 |
|------|------|------|
| Point 개수 | 변경 없음. 이미 Data 레이어에 Point로 존재함. | 없음 |
| 렌더링 방식 | 기본 위치 아이콘 → 작은 원(SVG) 아이콘으로 **스타일만 변경**. 마커 수·DOM 구조 동일. | 미미 |
| setStyle | Point일 때만 분기하여 icon 반환. 기존처럼 feature당 1회 호출. | 미미 |
| 클릭 리스너 | map.data에 click 리스너 1개 추가. | 미미 |
| 모달 | 1개, 클릭 시에만 표시. 평소 DOM 1개 추가 수준. | 미미 |

**결론**: 기능이 **현저히 느려지거나 끊김이 심해질 가능성은 낮음.**  
기존과 동일한 수의 Point가 같은 Data 레이어에 있으며, 아이콘만 바꾸고 클릭 시 모달만 띄우는 수준이라 추가 부하는 작음.
